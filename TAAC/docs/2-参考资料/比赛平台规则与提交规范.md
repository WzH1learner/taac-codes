# 比赛平台规则与提交规范

本文件是平台操作说明的中文精炼版，只保留和本项目训练、发布、评估直接相关的约束。更细的算法路线以 [推荐算法优化蓝图](../1-核心蓝图/推荐算法优化蓝图.md) 为准。

## 1. 比赛阶段

当前任务来自 TAAC2026 统一推荐挑战赛，第一轮比赛阶段为 2026-04-24 至 2026-05-23。我们的目标指标是官方 eval AUC。

## 2. 训练任务

平台训练任务通过上传脚本运行，训练模板必须包含：

```text
run.sh
```

训练任务启动时会自动执行 `run.sh`。本项目中 `train/run.sh` 会调用 `train/train.py`，并可通过平台附加参数覆盖默认值。

常用训练环境变量：

| 环境变量 | 用途 |
|---|---|
| `TRAIN_DATA_PATH` | 训练数据目录，包含 Parquet 与 `schema.json` |
| `TRAIN_CKPT_PATH` | checkpoint 输出目录 |
| `TRAIN_LOG_PATH` | 训练日志目录 |
| `TRAIN_TF_EVENTS_PATH` | TensorBoard 事件目录，当前不是核心依赖 |

训练 checkpoint 必须写入 `TRAIN_CKPT_PATH`。目录名建议以 `global_step` 开头，并只包含安全字符。

## 3. 模型发布

训练完成后，需要在训练任务实例的 Output 中选择 checkpoint 目录并发布模型。发布时应确认 checkpoint 目录至少包含：

```text
model.pt
schema.json
train_config.json
ns_groups.json  # 如果训练使用了分组文件
```

`train_config.json` 是 eval 重建模型结构的核心依据。缺失时，`eval/infer.py` 会使用 fallback 配置，容易造成 shape mismatch。

## 4. 官方 Eval

评估任务可从模型管理页或评估页创建。平台会调用 `eval/infer.py`。

常用 eval 环境变量：

| 环境变量 | 用途 |
|---|---|
| `MODEL_OUTPUT_PATH` | 已发布 checkpoint 目录 |
| `EVAL_DATA_PATH` | 官方测试数据目录 |
| `EVAL_RESULT_PATH` | 推理输出目录 |
| `EVAL_INFER_PATH` | 用户上传的推理脚本目录 |

`infer.py` 必须包含无参 `main()` 函数，并在 `EVAL_RESULT_PATH` 下生成：

```json
{
  "predictions": {
    "用户ID字符串": 0.1234
  }
}
```

注意事项：

- `predictions` 的 key 必须覆盖测试集有效 `user_id`，不能缺失或多出无关 ID。
- 输出概率应为 0 到 1 的浮点数。
- 结构性模型改动必须同步 `eval/model.py`、`eval/dataset.py` 和 `eval/infer.py`。

## 5. Eval 次数策略

平台每日 eval 次数有限。当前策略：

1. official eval 只提交变量清晰、训练健康、校准合理的候选。
2. 不为已否定路线重复消耗额度，例如 `time_window=2h`、`train_include_valid=True + last`、`emb_skip_threshold=7M`。
3. 每次 eval 后记录 AUC、推理耗时、checkpoint 配置和训练日志摘要。

## 6. 提交前检查清单

- `train/run.sh` active config 是否与本次实验目标一致。
- checkpoint 中是否有 `train_config.json`。
- 新增模型输入字段是否已同步到 eval。
- `predictions.json` 格式是否符合平台要求。
- 本次 official eval 是否回答一个明确问题。
