# Current Experiment Handoff - 2026-05-14

Official eval AUC is the source of truth. Valid AUC and calibration metrics are
used only to decide whether a checkpoint deserves official eval.

## 0. How To Read Runs Before Spending Eval

The short-term goal is to improve official eval AUC enough to advance from the
preliminary round. That means every cloud run should either produce an eval
candidate or remove uncertainty from the search space.

Do not only look at official eval AUC or a single best valid AUC line. Training
metrics are a diagnostic panel:

```text
valid AUC     = ranking/separation
LogLoss       = probability quality under confidence penalty
Brier         = calibration / squared probability error
prob_mean     = predicted base rate
label_mean    = true base rate on the validation split
prob_std      = score spread; useful within a run, not a universal target
```


Decision rule:

```text
1. Official eval AUC is the final winner.
2. Valid AUC says whether to spend official eval.
3. LogLoss/Brier/prob_mean decide whether a near-tie deserves to stay alive.
4. prob_std is a context signal, not a target. Higher is only good when ranking
   and calibration also improve.
5. Failed official experiments still matter if their metrics explain why the
   idea failed.
6. For time/history features, do EDA first, then train one clean variable.
```

implemented, not every possible time signal. Future time work should be

recent-match stability, and train-tail vs valid-tail drift checks.
