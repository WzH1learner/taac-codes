二、腾讯Angel机器学习平台操作指南
平台核心模块分为模型训练、模型发布与管理、模型评估三大类，操作说明如下：
（一）模型训练模块

创建训练任务流程

步骤：填写任务名称（Job Name）、任务描述（Job Description）→ 选择「本地上传」上传脚本 / 点击「新建脚本」在线编写 → 确认后点击「提交」保存任务。
注意：训练模板必须包含run.sh文件作为执行入口，任务启动时会自动运行该文件。
资源示例：单任务可选配置为1个GPU、4核CPU、16GB内存，训练数据集可选Tencent NLP等官方数据集。

训练任务管理

任务列表可查看Job ID、任务名称、描述、实例状态、最后更新时间，支持点击「Instances」查看任务输出与日志，点击「停止」终止运行中的任务，点击「More」可复制、删除、编辑任务。

训练环境配置

硬件环境：采用虚拟化GPU资源分配，单GPU分区规格为：20%单GPU计算能力、19GiB GPU内存、9核CPU、55GiB运行内存。
软件环境：Ubuntu 22.04系统，预装cuda 12.6、cudnn 9.5.1、cublas 12.6.3.3。

训练环境变量说明
平台将核心目录以环境变量形式传入运行容器，可直接在脚本中调用：
环境变量用途USER_CACHE_PATH20GB用户缓存路径，训练、评估阶段通用，可共享数据TRAIN_DATA_PATH训练数据集存储路径TRAIN_CKPT_PATH模型检查点（ckpt）保存路径TRAIN_TF_EVENTS_PATHTensorBoard事件文件存储路径

训练输出规范

模型权重必须保存到TRAIN_CKPT_PATH目录；
迭代检查点需放在以global_step为前缀的目录中，目录名仅允许包含字母、数字、下划线、连字符、等号、句点，长度不超过300字符，可在目录名附带训练参数（如迭代步数、学习率、层数等，示例：global_step20.lr=0.001.layer=2）；
TensorBoard仅支持标量指标，需通过TRAIN_TF_EVENTS_PATH初始化SummaryWriter，示例代码：
pythonfrom torch.utils.tensorboard import SummaryWriter
writer = SummaryWriter(os.environ.get('TRAIN_TF_EVENTS_PATH'))

（二）模型发布与管理模块

模型发布三步流程
① 训练任务页点击对应任务的「Instances」→ ② 进入实例页点击「Output」→ ③ 选择要发布的ckpt文件，点击「Publish」，填写模型名称、模型描述后提交即可完成发布。
模型管理页可查看所有已发布模型，点击模型卡片底部的「Model Evaluation」按钮，可直接进入该模型的评估流程。

（三）模型评估模块

评估任务创建入口（两种方法）

方法1：模型管理页选中目标模型，点击卡片底部「Model Evaluation」；
方法2：直接进入「模型评估」页，点击「Create Evaluation」新建任务。

评估环境变量说明
环境变量用途USER_CACHE_PATH同训练，20GB共享缓存，可跨阶段复用数据MODEL_OUTPUT_PATH模型输出路径EVAL_DATA_PATH推理用测试数据集目录EVAL_RESULT_PATH中间结果、最终预测文件predictions.json的输出路径EVAL_INFER_PATH用户上传推理脚本的目录

推理脚本规范

推理脚本必须命名为infer.py，且包含无参数的main()函数，否则评估失败；
若需安装自定义依赖，可上传命名为prepare.sh的脚本，系统会在推理开始前，在预激活的Conda环境中自动运行该脚本。

预测结果规范

main()函数必须生成predictions.json文件，保存到EVAL_RESULT_PATH目录；
文件格式要求：
json{
    "predictions": {
        "用户ID字符串": 0-1的转化概率浮点数,
        ...
    }
}

注意：predictions映射的所有键必须是测试集内的有效用户ID，不能缺失也不能多出无关ID，否则会影响最终得分。

评估任务管理

评估任务列表可查看Evaluation ID、任务名称、评估状态（等待资源/推理中/成功/失败等）、排行榜分数、推理时间、最后更新时间，支持查看排行榜、日志、复制任务等操作。
