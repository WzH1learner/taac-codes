# 序列建模与推荐系统技术汇总

## 一、序列推荐核心架构

### 1.1 Transformer-based 序列编码

```
输入序列: [e1, e2, e3, ..., eN]  (用户行为历史)
         ↓
    Embedding Lookup + Position Encoding
         ↓
    Multi-Head Self-Attention × L
         ↓
    FFN (SwiGLU / ReLU)
         ↓
    序列表征: [h1, h2, ..., hN]
```

**关键变体**：
- **标准 Transformer**：SASRec, BERT4Rec
- **因果注意力**：GPT-style，从左到右建模
- **双向注意力**：BERT-style，全序列可见

### 1.2 长序列处理技术

| 技术 | 原理 | 适用场景 |
|---|---|---|
| **Top-K 截断** | 只保留最近 K 个交互 | 实时性强的场景 |
| **分层注意力** | 先粗粒度聚类，再细粒度注意力 | 超长序列（>1000） |
| **LSH 注意力** | 局部敏感哈希，近似全局注意力 | 内存受限 |
| **LongerEncoder** | Top-K 压缩 + 稀疏注意力 | 腾讯 baseline 采用 |
| **Memory Network** | 外部记忆槽存储历史摘要 | 需要全局历史 |

### 1.3 时间感知建模

- **绝对时间编码**：将时间戳直接嵌入
- **相对时间编码**：计算相邻行为的时间间隔
- **周期性编码**：识别日/周/月周期模式
- **Time Bucket（时桶）**：将时间差离散化为桶（baseline 采用）

## 二、特征交互技术

### 2.1 特征交叉方法

| 方法 | 复杂度 | 效果 | 代表模型 |
|---|---|---|---|
| **内积** | O(d) | 捕获二阶交互 | FM, DeepFM |
| **外积/张量** | O(d²) | 高阶交互 | FFM, xDeepFM |
| **Attention** | O(n²d) | 自适应权重 | AutoInt, FiBiNET |
| **MLP** | O(ndh) | 隐式高阶 | DCN, PNN |
| **HyFormer 交叉注意力** | O(T²d) | 序列↔非序列统一 | PCVRHyFormer |

### 2.2 RankMixer 分析（baseline 采用）

```
Token 序列: [NS_token1, NS_token2, ..., Query_token1, Query_token2, ...]
                ↓
        RankMixerBlock
                ↓
    1. Token Mixing: 所有 token 做自注意力（特征交互）
    2. Per-Token FFN: 每个 token 独立做 SwiGLU
                ↓
    输出: 融合后的 token 表征
```

**三种模式**：
- `full`: token mixing + per-token FFN（完整交互）
- `ffn_only`: 只做 per-token FFN（轻量级）
- `none`: 恒等映射（无交互）

## 三、统一建模思路

### 3.1 序列 + 非序列统一

**核心问题**：如何在一个模型里同时处理行为序列（变长）和属性特征（固定）？

**方案对比**：

| 方案 | 做法 | 优点 | 缺点 |
|---|---|---|---|
| **拼接 MLP** | 序列 pooling 后 concat 所有特征 | 简单 | 丢失序列时序信息 |
| **双塔** | 序列塔 + 属性塔分别编码后交互 | 解耦 | 交互不充分 |
| **HyFormer** | 序列 Query + 非序列 NS Token 交叉注意力 | 充分交互 | 计算量较大 |
| **生成式** | 直接生成推荐内容 | 端到端 | 训练难度大 |

### 3.2 HyFormer 设计分析

```
序列域: A, B, C, D (4 个行为域)
        ↓ Seq Encoder (Transformer)
序列 Query: [qA1, qA2, qB1, qB2, ...] (num_queries × num_domains)

非序列: user_int, user_dense, item_int, item_dense
        ↓ NS Tokenizer (RankMixer / Group)
NS Token: [ns_u1, ns_u2, ..., ns_i1, ns_i2, ...]

               ↓ 交叉注意力
        HyFormer Block × N
               ↓
        所有 Token 互相交互
               ↓
        Classifier → pCVR
```

**设计亮点**：
1. Query 机制：每个域生成多个 query，捕获多视角信息
2. NS Tokenizer：将异构非序列特征压缩为统一 token
3. 交叉注意力：序列 ↔ 非序列充分交互
4. 分层处理：先域内编码，再全局交互

## 四、损失函数与优化

### 4.1 常用损失函数

| 损失 | 公式 | 适用场景 |
|---|---|---|
| **BCE** | -[y·log(p) + (1-y)·log(1-p)] | 二分类基础 |
| **Focal Loss** | -(1-p)^γ · log(p) | 类别不平衡 |
| **Pairwise** | max(0, margin + s_neg - s_pos) | 排序场景 |
| **Listwise** | 交叉熵(理想排序, 预测排序) | 全列表优化 |

### 4.2 优化策略

- **双优化器**：Adagrad(Embedding) + AdamW(Dense)
- **学习率差异**：Sparse lr=0.05, Dense lr=1e-4（稀疏特征更新更快）
- **冷重启**：每 epoch 重置高基数 embedding（防止过拟合）
- **早停**： patience=5，监控 AUC

## 五、工业级实践技巧

### 5.1 特征处理
- **ID 特征**：Hash 分桶，控制 vocab_size
- **连续特征**：分桶离散化或直接归一化
- **序列特征**：截断/压缩/分层采样
- **时间特征**：差分、周期编码、时桶

### 5.2 训练优化
- **负采样**：对大规模数据采样负样本
- **梯度累积**：模拟大批量训练
- **混合精度**：FP16/FP32 混合
- **模型并行**：大 embedding 表分片

### 5.3 评估指标
- **AUC**：区分正负样本能力
- **LogLoss**：概率校准度
- **GAUC**：按用户分组的 AUC
- **NDCG/Hit@K**：排序质量
