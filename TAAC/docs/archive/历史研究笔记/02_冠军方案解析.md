# TAAC2025 冠军方案技术解析（参考价值）

> 注意：以下分析基于 2025 年冠军方案，2026 年赛题方向已升级为"统一推荐"，但核心技术思路有继承性。

## Echoch 团队（冠军）方案

### 核心架构：三级会话体系
- 将用户行为序列组织为 **三级会话结构**
- 通过周期编码理解用户实时需求
- 引入随机性解决冷启动问题

### 关键技术点

| 技术 | 说明 |
|---|---|
| **三级会话体系** | 按时间/语义将序列分层建模，不同粒度捕获用户意图 |
| **周期编码** | 识别用户行为的周期性模式（如每周/每月购买习惯） |
| **冷启动随机性** | 引入随机扰动，为新用户/新物品提供探索能力 |
| **全模态生成式** | 端到端生成推荐内容，而非传统排序打分 |

### 创新点
- 不是传统"检索-排序-漏斗"，而是直接生成推荐结果
- 多模态信息（文本、图像、协同行为）统一编码
- 考虑时间维度的新鲜度特征

## leejt 团队（亚军）方案

### 核心架构：工程驱动的大规模训练

### 关键技术点

| 技术 | 说明 |
|---|---|
| **共享词表** | 跨模态共享 embedding 词表，减少参数量 |
| **哈希编码** | 处理超大规模离散特征，避免 vocab 爆炸 |
| **分布式训练优化** | 工程层面极致优化，支撑大规模模型扩展 |
| **多模态融合** | 文本、视觉、协同行为统一表征学习 |

### 工程亮点
- 解决超大规模数据下的内存和计算瓶颈
- 模型扩展性极强，工业级可落地

## 对我们的启发

### 1. 序列建模方向
- **长序列处理**：考虑 Top-K 压缩（LongerEncoder）、分层注意力
- **时间感知**：引入时间桶（time bucket）编码，捕获周期性行为
- **多域融合**：4 个 domain（A/B/C/D）序列分别编码后统一交互

### 2. 特征工程方向
- **冷启动处理**：对高基数特征做哈希或降维
- **交叉特征**：user/item 特征的组合交互
- **多模态嵌入**：如果后续有文本/图像特征，需要统一 embedding 空间

### 3. 模型结构方向
- **统一架构**：HyFormer 的交叉注意力 + RankMixer 就是统一建模的思路
- **生成式 vs 判别式**：当前 baseline 是判别式（BCE 分类），未来可探索生成式
- **参数效率**：共享词表、embedding 压缩（rankmixer tokenizer）

### 4. 工程优化方向
- **分布式训练**：DataLoader 多 worker、梯度累积
- **稀疏参数优化**：Adagrad 处理 Embedding，AdamW 处理 Dense
- **检查点策略**：cold-restart trick（高基数 embedding 每 epoch 重置）
