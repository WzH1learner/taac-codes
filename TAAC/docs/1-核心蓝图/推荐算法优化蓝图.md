# 推荐算法优化蓝图

更新时间：2026-05-13

目标：围绕 TAAC 2026 pCVR official eval AUC 做可归因、可复现的单变量实验。官方 eval 是最终判断，valid AUC 只用于筛选候选。

## 1. Current Best

```text
official eval AUC = 0.818095
experiment = D01_grouped_user_dense_single_token
```

当前主线：

```text
transformer + rankmixer + 5M + BCE + short seq + grouped user_dense + no time_context
```

旧 best：

```text
0.809921 = historical swiglu + rankmixer + 5M + BCE + short seq
```

`0.809921` 现在只是 fallback 记录，不再是主线。

## 2. 平台与代码约束

- 上传训练包时必须保证 `train/` 与 `eval/` 中的 model/dataset 同步。
- 任何结构参数必须写入 `train_config.json`，eval 从 checkpoint 读取。
- D01/R01 的 `T=16, d_model=64, 64 % 16 = 0`，不能随意新增 RankMixer token。
- official test 时间窗很窄，已观察到 `timestamp_span_hours≈1.55`，样本集中在北京时间 7/8/9 点。绝对 hour/weekday/root timestamp bucket 有分布偏移风险。
- 不基于 sample_data 判断涨分。

## 3. 主模型路径

```text
user/item int + user_dense -> NS tokens
seq_a/b/c/d -> seq tokens + time bucket embedding
NS tokens + seq mean -> MultiSeqQueryGenerator -> query tokens
query tokens + NS tokens -> MultiSeqHyFormerBlock -> RankMixer -> output_proj -> classifier
```

### D01 grouped user_dense

D01 保持 1 个 `user_dense` token，不改变 RankMixer token 数。核心是按 fid 分支处理 dense：

| Group | Fids | 处理 |
| --- | --- | --- |
| ue | `61` | 独立 MLP |
| semantic | `87` | 独立 MLP |
| stats | `62,63,64,65,66` | `sign*log1p(abs)` + clamp + LayerNorm + MLP |
| small | `89,90,91` | 独立 MLP |
| other | 未知 dense | 独立 MLP |

结论：`flat -> grouped` 是当前最高收益 verified change。

### R01 seq_recent_stats residual

R01 不复用失败的 T01 绝对/root time_context，也不新增 RankMixer token。它只在 D01 的 `final_repr` 上做 residual：

```text
final_repr = final_repr + seq_recent_stats_gate * MLP(seq_recent_stats)
```

每个样本 `seq_recent_stats_dim=28`，来自 4 个 domain * 7 维：

```text
len_log
last_gap_log
mean_gap_log
recent_count_30m
recent_count_2h
recent_count_1d
recent_ratio_2h
```

R01 的唯一训练变量：

```text
use_seq_recent_stats: 0 -> 1
seq_recent_stats_gate_init = 0.1
```
