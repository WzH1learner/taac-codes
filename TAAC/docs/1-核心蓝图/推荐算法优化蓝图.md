# 推荐算法优化蓝图

更新时间：2026-05-13

目标：围绕 TAAC 2026 pCVR official eval AUC 做可归因、可复现的单变量实验。官方 eval 是最终判断，valid AUC 只用于筛选候选。

## 1. Current Best

```text
official eval AUC = 0.818095
experiment = D01_grouped_user_dense_single_token
```

当前主线：

```text
transformer + rankmixer + 5M + BCE + short seq + grouped user_dense + no time_context
```

旧 best：

```text
0.809921 = historical swiglu + rankmixer + 5M + BCE + short seq
```

`0.809921` 现在只是 fallback 记录，不再是主线。

## 2. 平台与代码约束

- 上传训练包时必须保证 `train/` 与 `eval/` 中的 model/dataset 同步。
- 任何结构参数必须写入 `train_config.json`，eval 从 checkpoint 读取。
- D01/R01 的 `T=16, d_model=64, 64 % 16 = 0`，不能随意新增 RankMixer token。
- official test 时间窗很窄，已观察到 `timestamp_span_hours≈1.55`，样本集中在北京时间 7/8/9 点。绝对 hour/weekday/root timestamp bucket 有分布偏移风险。
- 不基于 sample_data 判断涨分。

## 3. 主模型路径

```text
user/item int + user_dense -> NS tokens
seq_a/b/c/d -> seq tokens + time bucket embedding
NS tokens + seq mean -> MultiSeqQueryGenerator -> query tokens
query tokens + NS tokens -> MultiSeqHyFormerBlock -> RankMixer -> output_proj -> classifier
```

### D01 grouped user_dense

D01 保持 1 个 `user_dense` token，不改变 RankMixer token 数。核心是按 fid 分支处理 dense：

| Group | Fids | 处理 |
| --- | --- | --- |
| ue | `61` | 独立 MLP |
| semantic | `87` | 独立 MLP |
| stats | `62,63,64,65,66` | `sign*log1p(abs)` + clamp + LayerNorm + MLP |
| small | `89,90,91` | 独立 MLP |
| other | 未知 dense | 独立 MLP |

结论：`flat -> grouped` 是当前最高收益 verified change。

## 6. 下一步路线

### P0. 保护 D01

- `train/run.sh` active block 保持 D01 current best。
- 显式写出 `--seq_encoder_type transformer`。
- 新参数默认关闭，保证 `use_seq_recent_stats=0` 等价于 D01。

### P1. D02_grouped_dense_swiglu

状态：已开启。

只改：

```text
seq_encoder_type: transformer -> swiglu
```

目的：验证 D01 grouped dense 是否能和历史 swiglu encoder 叠加。

### P2. R01_seq_recent_stats_residual_v1

今晚第二个训练优先级。

只改：

```text
use_seq_recent_stats: 0 -> 1
seq_recent_stats_gate_init = 0.1
```

设计原则：

- 不复用 T01 接法。
- 不使用 weekday/hour/root timestamp bucket。
- 不把时间特征加到 user_dense token。
- 不新增 RankMixer token。
- 只在 `final_repr` 上做 residual：`final_repr + gate * MLP(seq_recent_stats)`。

28 维 `seq_recent_stats` 来自 4 个 domain * 7 维：

```text
len_log
last_gap_log
mean_gap_log
recent_count_30m
recent_count_2h
recent_count_1d
recent_ratio_2h
```

判定：

```text
R01 best valid AUC >= 0.864399:
  候选 official eval。
R01 valid 接近 D01 且 LogLoss/Brier/prob_mean 更健康:
  保留候选。
R01 valid 明显低或校准崩:
  不 eval。
如果 D02 valid 更好:
  优先 D02。
```

### P3. P2_pair_target_history_match_eda

只做分析，不训练。

脚本：

```text
research/code/pair_match_eda.py
```

目标：验证 `item_int fid x seq sideinfo fid` 是否存在 target-history exact match 信号，包括 match rate、recent 2h match、matched gap、label lift。

只有 EDA 报告显示非极端 match rate 且 lift 有价值，才考虑 `P2_pair_dense_v1`。

### P4. S01_D01_seed_sweep

低成本候选，放在 D02/R01 后。

```text
seeds = 42, 2026, 3407, 1234, 2025
```

原则：

- 基线使用 D01 或 D02/R01 中表现最好的主线。
- 只改 `--seed`。
- 不改任何结构和训练参数。
- 先看 valid AUC、LogLoss、prob_mean/label_mean。
- 只有 valid 明显超过当前主线，才考虑 official eval。

注意：CUDA / SDPA / IterableDataset 可能导致非完全确定性，seed sweep 只能作为微调收益，不能替代结构性涨点。不要把 seed sweep 和 R01/hash/G01 混在同一次实验。

## 8. Official Eval 准入

官方 eval 前检查：

1. 单变量是否清楚。
2. valid AUC 是否接近或超过当前主线。
3. LogLoss/Brier 是否没有明显变坏。
4. `prob_mean` 是否没有远离 `label_mean`。
5. checkpoint 是否包含 `model.pt`、`train_config.json`、`schema.json`。
6. train/eval model 和 dataset 是否同步。
