# Recommendation Optimization Blueprint - TAAC 2026

Last updated: 2026-05-16

The only scoreboard that matters is official eval AUC. Valid metrics are used
to filter candidates, not to declare a winner.

## 0. Experiment Design Mindset

The near-term goal is preliminary advancement, so the optimization loop should
maximize useful signal per cloud run. A run is useful if it either improves
official eval AUC, creates a credible eval candidate, or teaches us which
direction to stop spending on.

Do not design experiments from valid AUC alone. Read the whole validation panel:

```text
valid AUC  -> ranking and separation
LogLoss    -> probability quality under confidence penalty
Brier      -> calibration error
prob_mean  -> predicted base rate
label_mean -> true validation base rate
prob_std   -> output spread; compare carefully
```

`prob_std` needs context. Inside one stable run family, higher `prob_std` can
track better separation. Across different feature/model changes, higher is not
automatically better: it can also mean confidently wrong predictions. A safer
read is:

```text
high AUC + stable/better LogLoss + healthy prob_mean = good separation
near-tie AUC + better LogLoss/Brier/prob_mean = keep as candidate
low prob_mean + low prob_std = conservative underprediction
high prob_std + bad LogLoss = overconfident wrong direction
```

EDA-before-training rule:

```text
For time, pair, and target-history ideas, run EDA first.
Check lift, coverage, count stability, train-tail vs valid-tail stability, and
official-window drift risk before spending a full training run.
Train only one variable at a time after the EDA explains why it should work.
```

## 2. Model Shape Guardrails

```text
RankMixer T = num_queries * num_sequences + num_ns
D01 T = 2 * 4 + 8 = 16
d_model = 64
64 % 16 = 0
```

Do not add new RankMixer tokens unless T and `d_model % T` are recalculated.

Train/eval synchronization is mandatory:

```text
train/model.py <-> eval/model.py
train/dataset.py <-> eval/dataset.py
train_config.json must drive eval/infer.py
```

## 4. Accepted Insight

Grouped user_dense is the strongest verified structural improvement.

The grouped projector keeps one final user_dense token, so RankMixer T does not
change. Dense fids are split into branches:

| Group | Fids | Handling |
| --- | --- | --- |
| ue | `61` | Branch MLP |
| semantic | `87` | Branch MLP |
| stats | `62,63,64,65,66` | `sign*log1p(abs(x))`, clamp, LayerNorm, MLP |
| small | `89,90,91` | Branch MLP |
| other | unknown dense fids | Branch MLP |

## 8. Pair EDA v2 Command

```bash
mkdir -p "${TRAIN_CKPT_PATH:-research/reports}"
python3 -u research/code/pair_match_eda.py \
  --data_path "${TRAIN_DATA_PATH:-/data_ams/academic_training_data}" \
  --max_files 1000 \
  --max_rows 200000 \
  --recursive_files 1 \
  --write_csv 1 \
  --summary_top_k 30 \
  --min_match_count 50 \
  --min_recent_match_count 10 \
  --top_k 50 \
  --focus_domain seq_d \
  --focus_side_fid 25 \
  --split_by_valid_tail 1 \
  --output "${TRAIN_CKPT_PATH:-research/reports}/pair_match_eda_v2.md" \
  2>&1 | tee "${TRAIN_CKPT_PATH:-research/reports}/pair_match_eda_v2.log"
```

Do not cat the full markdown in cloud logs:

```bash
REPORT="${TRAIN_CKPT_PATH:-research/reports}/pair_match_eda_v2.md"
awk '/^## Compact Summary/{flag=1} /^## Schema Candidates/{flag=0} flag{print}' "$REPORT"
awk '/^## Stable Candidate Pairs/{flag=1; n=0} /^## Unstable High-Lift/{flag=0} flag && n++ < 45{print}' "$REPORT"
awk '/^## Focus Table/{flag=1; n=0} /^## Top Pairs By Lift/{flag=0} flag && n++ < 45{print}' "$REPORT"
awk '/^## First 90% vs Tail 10% Stability/{flag=1; n=0} /^## Recommendations/{flag=0} flag && n++ < 45{print}' "$REPORT"
```

Stable filter:

```text
match_count_abs >= 50
match_any_rate >= 0.005
lift > 1.05
positive_rate_when_match > positive_rate_when_no_match
```

## 9. Checkpoint Selection Plan

Later independent experiment only:

```text
--keep_top_k_checkpoints 3
--checkpoint_select_metric auc_then_logloss
```

Rule:

```text
best_auc = max(valid_auc)
candidates = valid_auc >= best_auc - 0.0003
choose lower LogLoss
then lower Brier
then prob_mean closer to label_mean
```

Default remains `checkpoint_select_metric=auc`.
